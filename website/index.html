Git, without the ceremony.

SGIT is a small Rust wrapper around the Git CLI. It keeps Git’s defaults and config intact, but offers a few sharp, everyday commands with nicer ergonomics.

Quick start
sgit init
sgit stage .
sgit commit -m "first commit" --push
What it is
  • Fast pass-through to git
  • Fewer flags for common flows
  • Predictable behavior (no surprises)

Install

Build the binary and put it on your PATH. The repo includes convenience scripts.

Build
cd sgit
cargo build --release
Install / uninstall
./install.sh
./uninstall.sh

Set SGIT_INSTALL_DIR to install somewhere other than $HOME/.local/bin.

Usage

SGIT sticks to the shape of Git: a command, then options. Most commands forward directly to Git with a slightly smoother default. 

sgit <command> [options]

Commands

Command
What it does
init
Runs git init
stage [path...]
Adds files (defaults to .)
unstage [path...]
Runs git restore --staged
status [--short]
Shows git status (or -sb)
log [--short]
Compact or detailed log
diff [path] [--staged]
Diff working tree (or staged snapshot)
branch
Lists local branches
push [remote] [branch]
Pushes using Git defaults; override if needed
pull [remote] [branch]
Pulls with optional remote/branch
commit ...
Commit helpers: stage, push, amend
Commit helpers

SGIT can stage before committing, and optionally push right after. It still respects Git’s upstream settings and push.default. 

sgit commit -m "message" [--all | --unstaged | --staged] [--push] [--amend]

Explain mode

Add --explain to get a friendly, beginner-style explanation of the command and common options, instead of running it. 

sgit status --explain
sgit --explain
sgit is intentionally small. If a command isn’t here, use git.
